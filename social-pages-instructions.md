# Crawford Coaching Mailer — Social Sharing Pages Implementation

**Purpose:** Extend the renderer to automatically generate a social sharing page for
each food section (Body, Thought, Brain, Soul) at build time. Each page is a static
HTML file uploaded to the edition's Supabase Storage folder. The public URL of each
page becomes the `share_url` for that section, which the newsletter template renders
as a "Found this useful? Share it →" link.

---

## Architecture overview

```
renderer.py (build time)
    │
    ├─ For each food section with copy:
    │   1. Call Anthropic API → generate social post text
    │      (includes #crawfordcoaching + 4 context-specific hashtags)
    │   2. Build sharing page HTML:
    │      - Section image with download link
    │      - Post text with copy button
    │      - LinkedIn, Facebook, X share buttons
    │      - Open Instagram button
    │      - Subscribe CTA
    │   3. Upload to Supabase: newsletters/{slug}/socials/{section}.html
    │   4. Set share_url = public URL of that page
    │
    └─ Continue with normal render (share_urls now populated → template renders links)
```

The sharing pages are generated **before** the template token replacement, so the
`share_url` values flow into the rendered newsletter naturally.

```python
# ── Supabase Storage upload ──────────────────────────────────────────────────

def _upload_to_supabase(bucket: str, path: str, content: str, content_type: str = "text/html") -> str:
    """
    Uploads content to Supabase Storage and returns the public URL.
    Overwrites if the file already exists.
    Requires SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY in the environment.
    """
    import urllib.request
    import urllib.error

    base = os.getenv("SUPABASE_URL", "").rstrip("/")
    key = os.getenv("SUPABASE_SERVICE_ROLE_KEY", "")
    if not base or not key:
        raise EnvironmentError(
            "SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY are required for upload"
        )

    upload_url = f"{base}/storage/v1/object/{bucket}/{path}"
    data = content.encode("utf-8")

    req = urllib.request.Request(
        upload_url,
        data=data,
        method="PUT",
        headers={
            "Authorization": f"Bearer {key}",
            "Content-Type": content_type,
            "x-upsert": "true",
        },
    )

    try:
        urllib.request.urlopen(req)
    except urllib.error.HTTPError as e:
        body = e.read().decode("utf-8", errors="replace")
        raise RuntimeError(f"Supabase upload failed ({e.code}): {body}")

    return f"{base}/storage/v1/object/public/{bucket}/{path}"
```

## 2. Add social post generation function

Add this function after the existing `_proofread` / `_maybe_proofread` functions.

**Important:** The prompt specifies `#crawfordcoaching` as a mandatory hashtag plus
exactly 4 context-specific hashtags generated by the model.

```python
# ── Social post generation ───────────────────────────────────────────────────

def _generate_social_post(
    section_title: str,
    section_subtitle: str,
    section_copy: str,
    blog_url: str,
) -> str:
    """
    Generates a social media post for a newsletter section using the Anthropic API.
    Returns a single post suitable for LinkedIn, Facebook, Twitter/X, and Instagram.
    Always includes #crawfordcoaching plus 4 context-specific hashtags.
    Requires ANTHROPIC_API_KEY in the environment.
    """
    import anthropic

    client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

    # Strip HTML tags from the copy to give the model clean text
    clean_copy = re.sub(r"<[^>]+>", "", section_copy).strip()

    message = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=512,
        messages=[
            {
                "role": "user",
                "content": (
                    "Write a social media post to share the following newsletter section. "
                    "The post should be suitable for LinkedIn, Facebook, Twitter/X, and Instagram. "
                    "Keep it under 200 words. Use a thoughtful, reflective tone that matches "
                    "a coaching and wellness brand. Do not use excessive emojis — one or two "
                    "at most is fine.\n\n"
                    "At the end of the post, include exactly 5 hashtags on their own line:\n"
                    "- #crawfordcoaching (always first, always included)\n"
                    "- Plus 4 hashtags relevant to the specific content of this section\n\n"
                    "After the hashtags, end with the blog link on its own line.\n\n"
                    "Do NOT include any preamble, explanation, or options. Return ONLY the "
                    "post text, ready to copy and paste.\n\n"
                    f"Section: Food for the {section_title}\n"
                    f"Subtitle: {section_subtitle}\n"
                    f"Content:\n{clean_copy}\n\n"
                    f"Blog link: {blog_url}"
                ),
            }
        ],
    )
    return message.content[0].text.strip()
```

        image_block = (
            '<div class="section-image">'
            f'<img src="{escape(image_url)}" alt="{escape(image_alt)}">'
            f'<a class="save-image" href="{escape(image_url)}" '

        tweet_encoded=quote(tweet_text, safe=""),
        subscribe_url=escape(subscribe_url or "https://crawford-coaching.ca/subscribe"),
        year=datetime.now().year,
    )

## 4. Add the social generation orchestrator

Add this function, which ties together generation, page building, and upload.
Note that it resolves the section image URL and passes it to the page builder.

```python
# ── Social sharing orchestrator ──────────────────────────────────────────────

SECTION_MAP = {
    "food_body":    ("Body",    "body"),
    "food_thought": ("Thought", "thought"),
    "food_brain":   ("Brain",   "brain"),
    "food_soul":    ("Soul",    "soul"),
}


def _generate_share_pages(
    data: dict[str, Any],
    slug: str,
) -> dict[str, str]:
    """
    Generates social sharing pages for all four food sections.
    Uploads each page to Supabase Storage.
    Returns a dict mapping section keys to their public share page URLs.

    Example return:
        {"food_body": "https://...newsletters/15-.../socials/body.html", ...}
    """
    blog_url = _str(data.get("full_blog_url"))
    subscribe_url = _str(data.get("subscribe_url"))
    share_urls: dict[str, str] = {}

    for section_key, (title, filename) in SECTION_MAP.items():
        section = data.get(section_key) or {}
        copy = _str(section.get("copy"))
        subtitle = _str(section.get("subtitle"))

        if not copy:
            continue

        # Resolve the section image to a full URL for the sharing page
        image_url = _resolve_image(_str(section.get("image")))
        image_alt = _str(section.get("image_alt"))

        print(f"  Generating social post for {title}...")
        post_text = _generate_social_post(
            section_title=title,
            section_subtitle=subtitle,
            section_copy=copy,
            blog_url=blog_url,
        )

        page_html = _build_share_page(
            section_title=title,
            section_subtitle=subtitle,
            post_text=post_text,
            blog_url=blog_url,
            subscribe_url=subscribe_url,
            image_url=image_url,
            image_alt=image_alt,
        )

        # Also save the raw post text for future use
        supabase_path_html = f"{slug}/socials/{filename}.html"
        supabase_path_txt = f"{slug}/socials/{filename}.txt"

        url = _upload_to_supabase("newsletters", supabase_path_html, page_html, "text/html")
        _upload_to_supabase("newsletters", supabase_path_txt, post_text, "text/plain")

        share_urls[section_key] = url
        print(f"    → {url}")

    return share_urls
```

## 10. Hashtag specification

Every generated social post must include exactly 5 hashtags:

1. `#crawfordcoaching` — **always first, always present** (brand anchor)
2–5. Four hashtags generated by the AI, specific to the section content

Example output for the Body section ("Eat Around, Not On Top"):
```
#crawfordcoaching #nutrition #protein #healthyhabits #mealplanning
```

Example output for the Brain section ("One Question To Replace A Label"):
```
#crawfordcoaching #mindset #foodlabels #contextualthinking #wellbeing
```

This is enforced by the prompt in `_generate_social_post()`. If the model omits
`#crawfordcoaching` or produces too few/many hashtags, the post text is still
usable — this is a best-effort instruction, not a programmatic constraint.

## 12. Graceful degradation

The implementation degrades gracefully:

- **No `ANTHROPIC_API_KEY`**: Social pages are skipped, `share_url` stays empty,
  the share link in the newsletter is removed by the conditional processor.
  A console message confirms the skip.

- **API call fails**: The `try/except` wrapper catches any failure, logs a warning,
  and continues rendering without share links.

- **Supabase upload fails**: Same — caught, logged, rendering continues.

- **Section has no copy**: Skipped entirely (no API call, no upload, no share link).

- **Section has no image**: The sharing page renders without an image block.
  The post text, share buttons, and subscribe CTA all still work.

## 13. Usage

No new CLI flags needed. Social pages generate automatically:

```bash
# Standard render — generates social pages automatically
python renderer.py content/15-becoming-a-snacker.json

# With proofreading — social pages also generated
python renderer.py content/15-becoming-a-snacker.json --proofread

ANTHROPIC_API_KEY= python renderer.py content/15-becoming-a-snacker.json
```
