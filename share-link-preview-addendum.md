# Crawford Coaching Mailer — Share Link Preview Placeholder (Addendum)

**Purpose:** Show the "Found this useful? Share it →" link in the webapp preview
for all food sections that have copy, so the layout is visible during the design
phase. The link points to `#` (non-functional) — the real URLs are generated by
the CLI renderer at build time.

This is a single change to `templates.ts`.

---

## Update `renderNewsletterPreview()` in `templates.ts`

In the `replacements` object, the `*_SHARE_URL` values are currently set from the
content data, which is typically empty during authoring:

```typescript
BODY_SHARE_URL:      str(body.share_url),
THOUGHT_SHARE_URL:   str(thought.share_url),
BRAIN_SHARE_URL:     str(brain.share_url),
SOUL_SHARE_URL:      str(soul.share_url),
```

Replace those four lines with:

```typescript
BODY_SHARE_URL:      str(body.share_url) || (str(body.copy) ? "#" : ""),
THOUGHT_SHARE_URL:   str(thought.share_url) || (str(thought.copy) ? "#" : ""),
BRAIN_SHARE_URL:     str(brain.share_url) || (str(brain.copy) ? "#" : ""),
SOUL_SHARE_URL:      str(soul.share_url) || (str(soul.copy) ? "#" : ""),
```

**Logic:** If the content JSON already has a real `share_url` (post-render), use it.
Otherwise, if the section has copy (meaning a sharing page will be generated at build
time), show the link as `#` so it's visible in the preview. If the section has no copy,
the link stays hidden.

No other changes needed. The `BODY_SHARE_URL` flag in the flags object already
evaluates `Boolean(body.share_url)`, and since `"#"` is truthy, the conditional
block will render.
